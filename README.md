# 底稿系统 / Manuscript OS

**一份文件，让 AI 变成你的人。**

---

## 这是什么

底稿系统是一套方法论：给 AI 写一份关于你的「底稿」文件，让它**永久记住你是谁、你怎么想、你怎么说话、你每天在干什么**。

不是提示词。提示词是一次性的——每次对话说一遍，说完就忘。底稿是 AI 每次打开新对话自动读取的文件，你写一次，它永远记得。而且底稿是活的——**不是你手动改它，是 AI 自己在用的过程中不断更新它**，形成一个学习自循环。

---

## 我是谁，我为什么做这个

我叫舟舟，做内容创业，日更 500+ 天。一个人干产品、内容、销售、客服、运营。

我在用 GPT 的时候就已经摸出了一套让 AI 学习我风格的方法：**把我的历史内容、写作样本、风格数据全部喂给它，让它自己学**。效果不错，AI 确实能写出接近我风格的东西。

但 GPT 有一个致命问题：**它是对话型的，记忆有限**。对话一关，它就什么都忘了。下次打开，还要重新喂一遍数据、重新解释一遍规矩。我花了大量时间在"让 AI 想起我是谁"这件事上。

后来我找到了 Claude Code。它解决了 GPT 最大的短板：**本地读取文件 + AI 自己修改文件**。

这意味着什么？

我把底稿写成一份本地 Markdown 文件。AI 每次打开新对话自动读取这份文件——不需要我说任何话，它就已经知道我是谁、该干什么。更关键的是，**AI 在工作过程中会自己更新这份文件**：学了新东西，它自己加进去；发现新的模式，它自己标注；资产库有新条目，它自己编号存档。

这就形成了一个**学习自循环**：

```
底稿告诉 AI 你是谁
    ↓
AI 按底稿工作
    ↓
工作过程中产生新数据
    ↓
AI 自动把新数据写回底稿
    ↓
下次打开，AI 比上次更懂你
```

不是你手动维护 AI 的记忆，是 **AI 自己维护自己的记忆**。你只需要在它偏了的时候纠正一下方向。

## 我每天在用底稿做什么

### 场景一：第二大脑（跑了 500+ 天）

日记不是为了写爆款。日记是我的**数据采集入口**——每天口述几分钟，收集自己的故事、观点、情绪、决策过程。AI 把这些原始素材变成结构化的内容资产，存进我的第二大脑，方便我随时调用去做其它创作。

具体流程（全部写在底稿里，AI 自动执行）：

**采集：** 我口述，AI 整理。AI 知道我是口述的，会自动修正语音识别错误，但不改我的句式——底稿里写了"最小改动原则，保留语气和金句，不要重写"。整理完必须先发我审核，我说"可以"才进入下一步。

**提炼：** 审核通过后，AI 从这篇日记里自动提取 5 类资产：

- **故事**：场景→冲突→代价→反转→可复用的开头钩子。一条故事资产可以在文章、短视频脚本、课程案例里反复使用。
- **洞见**：反直觉 + 有解释力 + 来自真实经历的认知。每条洞见起一个 4 字以内的名字（Concept Handle），好传播好记忆。
- **风格表达**：标志性原句，攒多了就是语言指纹库。
- **动作**：读者 10 分钟内能执行的具体步骤。
- **母题**：对应哪个长期主题。当一个新观点在 3 篇以上日记里反复出现，AI 会提议新增母题。

500 多天下来，资产库积累了**200+ 条故事、40+ 条洞见、10+ 个母题**。

这些资产不是放着好看的。写长文时，AI 从故事库里调相关故事串起来；做短视频脚本，AI 用洞见库里的 Concept Handle 当钩子；设计课程，AI 按母题分类找出所有相关内容。

**日记是输入端，资产库是输出端，底稿是让这两端自动连接的系统。**

每次新对话，AI 读底稿就知道：当前各库编号到哪了、从哪接续、风格规矩是什么。不需要我说任何话。

**教练层：** AI 不只是整理员。每天给我教练反馈：今日观察（是突破还是在重复同一个困境）、做得好的地方、值得留意的地方、一个挑战问题。底稿里写了"不能只说好话，必须指出真实问题"。如果我连续聊同一个困境，AI 会直接说"你已经多次提到这个，有没有10分钟动作可以先做？"

### 场景二：情报雷达（每天自动跑）

我需要追踪行业动态——AI 工具更新、竞品动作、海外趋势。以前靠自己刷 Twitter、Product Hunt、GitHub，一刷就是一小时，看完还记不住。

现在这件事完全写在底稿里，AI 每天定时自动执行：

- 扫描指定信息源（Product Hunt、GitHub Trending、Reddit、HackerNews、中文科技媒体）
- 按我关心的领域分类过滤（AI 工具、内容创作、知识付费）
- 生成一份结构化的情报简报，存到固定目录
- 有重大变化时主动提醒我

我不用打开任何网站，每天早上读一份 AI 整理好的简报就行。底稿里定义了扫描源、过滤规则、输出格式、存储路径——换一个新对话，AI 读完底稿就知道该怎么跑。

**这是底稿系统"自动化"的一面：** 不只是你跟 AI 对话时它配合你，而是你不在的时候它也在按底稿里的规则替你工作。

### 场景三：学习系统（用 AI 学任何东西）

我用底稿系统学投资，零基础，11 天掌握了基础分析框架。

核心设计：**AI 是教练，不是顾问——数据给我看，让我自己推导，不直接给结论。**

底稿里有一个「已掌握框架」清单。**这个清单是 AI 自己更新的。** 每天学完，AI 自动把新掌握的框架写进底稿。第一天清单是空的，11 天后变成了 20 多个分析框架，覆盖宏观、行业、公司、估值四个层面。

每次打开新对话，AI 读到这个清单，就知道从哪接续——不重复教已会的，不跳过还没学的。

底稿还定义了学习阶段和完成标志。AI 知道我当前在哪个阶段，不会提前教后面的内容。

**底稿让"学习进度"变成了一个持久化的、AI 可读的状态文件。** 这套逻辑不限于投资——学任何东西都可以用同样的底稿结构。

## 底稿系统 vs 传统 AI 用法

大多数人用 AI 是这样的：

```
打开对话 → 说一段提示词 → AI 回复 → 关掉对话 → 下次重来
```

每次对话都是独立的。AI 不知道你上次做了什么，不知道你的风格偏好，不知道你的规矩。你花了 80% 的时间在"让 AI 进入状态"，只有 20% 的时间在干活。

底稿系统是这样的：

```
AI 自动读底稿 → 已经知道你是谁 → 直接开始干活 → 干完自动更新底稿 → 下次更懂你
```

区别不是效率高一点，是**完全不同的协作模式**。一个是每次雇一个新人从头培训，另一个是有一个跟了你很久的搭档，你不用说，它就知道该怎么做。

## 底稿是怎么工作的

底稿文件本质上就是一份 Markdown 文档，包含五个核心模块：

### 1. 你是谁（身份层）

不是写"内容创作者"就完了。要具体到 AI 能据此做判断：

```
内容创业者，日更 500+ 天
全职，一人多面（产品/内容/销售/客服/运营）
核心价值观：行动至上、反内耗
```

AI 读到这些，给建议就不会是"建议您考虑组建团队"——因为它知道你是一个人。

### 2. AI 的角色（关系层）

一句话定义 AI 的定位：

```
你是教练 + 整理员 + 内容生产线，不是助手。
```

"不是助手"很关键。不说的话 AI 默认助手模式——你说什么就做什么，不提反对意见。

### 3. 你的风格（表达层）

不是写"口语化"就够了，要具体到 AI 能精确模仿：

```
句式：口语化长句，逗号连接，一口气说完
态度：自嘲但不自卑，犀利但不刻薄
标志性表达："先给你们祛魅""好比说……好比说……"
禁区：不用鸡汤、不用学术术语、不用"首先其次最后"
```

禁区清单比正面描述更有效——告诉 AI "不能做什么"比"应该做什么"更精准。

### 4. 你的规矩（底线层）

每条规矩背后都是一次 AI 犯错后加上的：

```
不能加小标题
不能重写句子
不能删内容
文章必须先审核
审核通过后第一件事是存档
```

### 5. 工作流程（执行层）

用指令表定义——我说什么话，你做什么事：

```
「这是我的日记」→ 走完整处理流程
「整理一下」→ 只做标题 + 文章，等审核
「不用改，按原文」→ 原文存档，直接提取资产
「下一步」→ 继续后续步骤
```

一句话启动一整套流程，不用每次解释。

## 底稿的真正价值：AI 自己更新自己

很多"AI 记忆"方案是让**你**手动维护 AI 的知识库。底稿系统不一样——**AI 在工作过程中自己更新底稿**。

举几个具体例子：

**资产库自动增长：** 我口述完日记，AI 提取故事存进故事库，编号自动接续。我不需要自己去管"现在到第几条了"。

**学习进度自动记录：** 投资学习每次学完，AI 自动把新掌握的框架加进底稿的「已掌握」清单。下次新对话就从新的起点接续。

**母题自动识别：** 当一个观点在 3 篇以上日记中反复出现，AI 会标注"观察中"；出现 3 次以上，主动提议新增为正式母题。

**这意味着什么？** 你不是在"使用"一个工具，你是在**养**一个系统。它每天都在吸收你的数据、更新自己的认知、变得更懂你。用 30 天和用 300 天的底稿，是完全不同的东西。

## 怎么开始

### 第一步：选一个模板

从 `templates/` 目录选一个：

| 模板 | 适合谁 |
|------|--------|
| `blank.md` | 所有人。最精简骨架，10 分钟填完 |
| `content-creator.md` | 内容创作者。包含内容处理流程 + 资产提取 + 风格保护 |
| `learning-system.md` | 想用 AI 学任何东西的人。AI 教练 + 三阶段路线 + 自动进度追踪 |

### 第二步：喂数据

先把你已有的内容/数据给 AI 学习——历史文章、写作样本、风格参考、产品信息。AI 学完之后，帮你填充底稿里的风格描述和工作流程。

不要自己从零开始写风格描述。让 AI 看完你的数据之后，**由它总结你的风格**，你来确认和修正。这比你自己描述自己准确得多。

### 第三步：让 AI 读取

| 工具 | 怎么加载底稿 |
|------|-------------|
| **Claude Code** | 命名为 `CLAUDE.md`，放项目根目录，自动读取 |
| **Cursor** | 放 `.cursorrules` 或项目指令中 |
| **ChatGPT** | 粘贴到"自定义指令" |
| **豆包/Kimi** | 对话开头粘贴，说"先读完再回复" |

### 第四步：用、纠正、让它迭代

用的过程中 AI 一定会有不准的地方。纠正它，它会自己更新底稿。

- AI 写的不像你 → 告诉它哪里不对，让它把纠正点补进风格描述
- AI 做了你没要求的事 → 加一条规矩
- AI 漏了一个步骤 → 补进工作流程

关键心态：**底稿不是你写给 AI 的说明书，是你和 AI 共同维护的协作协议。**

## 常见问题

**Q：底稿要写多长？**
刚开始不用自己写太多——先把数据喂给 AI，让它帮你生成初版。然后在使用中不断纠正和补充。最终会长到 1000-3000 字，但每一行都是有用的。

**Q：我不用 Claude Code，能用吗？**
能。底稿就是一份文本文件。Claude Code 的优势是自动读取 + AI 能直接修改文件（自循环），其他工具需要手动粘贴，但核心逻辑一样。

**Q：跟 ChatGPT 的"记忆"功能有什么区别？**
ChatGPT 的记忆是它自己决定记什么，你控制不了。底稿是你完全可控的——你决定 AI 知道什么、不知道什么、按什么规矩工作。而且底稿是一份文件，可以备份、可以版本管理、可以跨工具迁移。

**Q：底稿能用在团队里吗？**
能。团队共享一份"项目底稿"，每个人再加自己的"个人底稿"。
